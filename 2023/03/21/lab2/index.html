实验题目：利用云平台搭建个人博客
实验学时：2 实验日期： 3/14
实验目的：熟悉个人博客系统的搭建。
具体包括：
参考方案：注册Github账号，搭建Hexo环境并实现个人博客搭建，撰写实验报告。

硬件环境：

联网的计算机一台

软件环境：

Windows

实验步骤与内容：

1、创建仓库

在github创建个人仓库，仓库名为用户名.github.io

2、安装git

3、git设置

在git bash里设置user.name and user.email

git config –global user.name “你的GitHub用户名”
git config –global user.email “你的GitHub注册邮箱”

4、将本地的git与github通过ssh连接

生成ssh密钥文件： ssh-keygen -t rsa -C “你的GitHub注册邮箱”
将生成的id_rsa.pub中的密钥复制，并在Github_Setting_Keys页面新建一个SSH key

5、安装node.js,检查node.js与npm安装成功

6、安装hexo

7、初始化博客

hexo init blog

8、初始化完成后会在当前目录下创建一个blog目录，再进入blog目录进行测试

9、在本地浏览器打开localhost:4000

10、将Hexo与Github关联起来，即打开站点的配置文件_config.yml修改

11、安装git部署插件

然后分别输入三条命令：

hexo clean
hexo g
hexo d

部署完成，已经可以访问了 Sdu-pjLiu.github.io

结论分析与体会：

1、Hexo常用命令：

npm install hexo -g #安装Hexo
npm update hexo -g #升级
hexo init #初始化博客

命令简写

hexo n “我的博客” == hexo new “我的博客” #新建文章
hexo g == hexo generate #生成
hexo s == hexo server #启动服务预览
hexo d == hexo deploy #部署

hexo server #Hexo会监视文件变动并自动更新，无须重启服务器
hexo server -s #静态模式
hexo server -p 5000 #更改端口
hexo server -i 192.168.1.1 #自定义 IP
hexo clean #清除缓存，若是网页正常情况下可以忽略这条命令

2、 还可以使用解析后的域名与个性化的主题使网站更加个性化

3、 如果将文章推送到博客上的话，可以使用hexo n “新文章名” ,再hexo g hexo d；这会将文件提交到master分支中，再将master中的分支合并到main中
