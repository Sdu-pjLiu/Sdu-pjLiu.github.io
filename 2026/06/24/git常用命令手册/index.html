Git 工作区与仓库关系、基础提交流程、分支同步、合并变基、贮藏与常用命令速查。

Git 是日常开发中最常用的版本控制工具。本文按「概念 → 基础操作 → 分支同步 → 进阶场景 → 速查」组织，便于查阅与复习。

一、Git 区域与对象关系

1.1 本地四个区域

Git 在本地维护四个相互关联的区域：

区域 说明 典型命令
工作区（Working Directory） 磁盘上实际编辑的文件 直接改文件
暂存区（Staging Area / Index） 下次提交的快照 git add
本地仓库（Local Repository） 已提交的历史与分支 git commit
贮藏区（Stash） 临时保存未提交的改动 git stash

1.2 本地仓库、远程仓库与跟踪分支

除上述四个区域外，还有三类「分支对象」需要区分：

对象 示例 说明
本地分支 main、feature_x 指向本地某次提交的可移动指针
远程仓库分支 origin/main、deepbi/feature_x 托管在 GitHub / GitLab 等服务器上的分支
远程跟踪分支 remotes/origin/main 本地缓存的远程分支状态，由 fetch 更新

关系示意：

  • push：把本地分支的提交推送到远程仓库。
  • pull：fetch + merge（或 rebase），拉取远程更新并合并到当前本地分支。
  • fetch：只更新远程跟踪分支，不改动工作区与当前分支。

十一、常用命令速查

场景 命令
更新远程状态 git fetch --all --prune
查看所有分支 git branch -a
查看分支关系 git branch -vv
暂存并提交 git add . && git commit -m "msg"
推送 git push / git push -u origin branch
拉取 git pull / git pull --rebase
删除本地分支 git branch -d branch_name
强制删除本地分支 git branch -D branch_name
删除远程分支 git push origin --delete branch_name
推送新分支 git push -u origin branch_name
跟踪远程分支 git switch -c branch_name origin/branch_name
贮藏改动 git stash / git stash pop
撤销工作区改动 git restore file
取消暂存 git restore --staged file
安全撤销已 push 提交 git revert commit_hash

十二、日常推荐习惯

  1. 提交前：git status + git diff，确认改动范围。
  2. 拉代码前：git fetch --all --prune，了解远程变化后再 merge / rebase。
  3. 分支清理：定期 git branch -vv，删除带 gone 的本地分支。
  4. 推送前：git log --oneline origin/main..HEAD，确认待推送的提交。
  5. 已共享分支：用 revert 而非 reset --hard 回退，避免改写他人已拉取的历史。
